User Manual for ​Interactive IoT-Based Learning and Conversational System for Nursery Children

The Nursery Learning System is designed to engage children in interactive audio-based learning. Using two ESP32 boards, the system allows children to ask questions or tap NFC tags to hear rhymes or songs. The system integrates Google Speech-to-Text, Chat GPT API, Google Text-to-Speech, and Firebase.

NFC Tags

The NFC tags used in the system are programmed with various Sinhala alphabets and numbers, helping children to learn and recognize them through interactive audio playback. Each NFC tag corresponds to a specific Sinhala letter or number. When a child taps an NFC tag to the reader, the system retrieves the corresponding audio file from Firebase and plays it through the speaker, allowing children to hear and learn the correct pronunciation of Sinhala letters and numbers in an engaging way.

 Tips For Use 

Power Bank

Ensure the power bank is fully charged before using the system for prolonged periods

Patient

The system may take a few moments to process voice inputs and NFC taps, so be patient while it retrieves and plays the content

Distance

For better voice recognition, position the microphone at an optimal distance (approximately 10-15 cm from the child’s mouth) when speaking

 Tips For What Not To Do 

Do Not Expose to Water

Avoid using the system near water or in damp environments, as the electronic components  can be damaged by moisture.

Do Not Tap NFC Tags Quickly

Avoid rapidly tapping the NFC tag against the reader. Hold it in place for a second or two to allow proper scanning

Do Not Shout into the Microphone

Speaking too loudly or shouting into the microphone may distort the audio input and lead to inaccurate speech recognition

Hardware Components Used in the​ System

How the Hardware Components Work Together

In this Nursery Learning System, the hardware components work together to create an interactive experience. The microphone captures the child's voice and sends the audio signal to the first ESP32 board, which connects to the Google Speech-to-Text API to convert the speech into text. This text is then processed by the ChatGPT API, and the response is converted back into speech using the Google Text-to-Speech API. The audio is sent to the second ESP32 board, which passes the signal to the I2S amplifier to amplify the sound for the speaker, allowing the child to hear the answer. For NFC-based learning, when a child taps an NFC tag to the NFC reader, the second ESP32 retrieves the corresponding rhyme or song from Firebase and plays it through the amplifier and speaker. Both ESP32 boards are powered by a power bank, ensuring mobility and seamless operation of the system.

Privacy and Security

Voice Data Handling

When a child speaks into the microphone, the audio is transmitted securely to Google's Speech-to-Text API. This transmission is encrypted, ensuring that the voice data is protected during its journey to and from the cloud. The system only processes audio temporarily for the purpose of generating a response and does not store or retain any voice recordings, ensuring that no personal data is kept.

Data Transmission

All communications between the ESP32 boards, Firebase, Google APIs (Speech-to-Text and Text-to-Speech), and ChatGPT API are secured using encrypted protocols like HTTPS. This ensures that any data, whether it's audio, text, or API responses, is securely transferred, preventing unauthorized access or interception.

No Personal Data Collection

The system does not collect or store any personal information about the children using it. All interactions are anonymous, and the data flow is limited strictly to processing audio inputs and providing learning responses.
